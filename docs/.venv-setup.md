# Virtual Environment Setup Guide

This guide explains how to set up a Python virtual environment for CSEL development.

## Prerequisites

### System Packages
Install required system packages first (cannot be installed via pip):

```bash
sudo apt-get update
sudo apt-get install python3-tk lsb-release
```

## Virtual Environment Setup

### 1. Create Virtual Environment

```bash
# Create virtual environment in .venv directory
python3 -m venv .venv
```

### 2. Activate Virtual Environment

```bash
# Activate the virtual environment
source .venv/bin/activate
```

You should see `(.venv)` prefix in your terminal prompt.

### 3. Install Python Dependencies

```bash
# Install all required packages
pip install -r requirements.txt
```

### 4. Verify Installation

```bash
# Check installed packages
pip list

# Test imports
python3 -c "import tkinter; import sqlalchemy; import psutil; print('All imports successful')"
```

## Development Workflow

### Start Development Session

```bash
# Activate virtual environment
source .venv/bin/activate

# Run configurator
sudo -E ./configurator
# Note: -E preserves environment variables including venv
```

### Build Binaries

```bash
# Ensure venv is activated
source .venv/bin/activate

# Build configurator
pyinstaller --onefile configurator.py

# Build scoring engine
pyinstaller --onefile scoring_engine.py
```

Built binaries will be in the `dist/` directory.

### Deactivate Virtual Environment

```bash
deactivate
```

## Common Issues

### "tkinter not found" error
- Make sure `python3-tk` system package is installed
- Verify with: `python3 -c "import tkinter"`

### "No module named 'sqlalchemy'" when running with sudo
- Use `sudo -E` to preserve environment variables
- Or install packages system-wide (not recommended)

### PyInstaller build includes wrong Python version
- Ensure venv is activated before running pyinstaller
- Check with: `which python3`

## Updating Dependencies

```bash
# Add new package to requirements.txt, then:
source .venv/bin/activate
pip install -r requirements.txt

# To freeze current versions:
pip freeze > requirements.txt
```

## Directory Structure

```
CSELv2/
├── .venv/              # Virtual environment (gitignored)
├── requirements.txt    # Python dependencies
├── .venv-setup.md      # This file
├── configurator.py     # Source code
├── scoring_engine.py   # Source code
├── dist/               # Built binaries (created by PyInstaller)
└── build/              # Build artifacts (created by PyInstaller)
```
